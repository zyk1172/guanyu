# 观隅付费与核验增强设计

## 目标

把当前“观隅”从单纯账号型新闻审视工具升级为带反滥用、额度消耗、点数购买、双搜索供应商和正式 Markdown 导出的可运营版本。

## 范围

本阶段实现：

- Markdown 导出格式升级，覆盖完整报告、原文、联网依据和提问记录。
- Tavily 与 Serper.dev 双搜索接入，账号级与管理员级 key 均可使用。
- 联网核验必须写出依据来源；无法核验时明确标注“暂无法核验”。
- 每用户每天 3 次免费额度，免费额度走管理员 API 与管理员搜索 key。
- 点数模式：6 元 30 点，快速审视消耗 1 点，深度审视消耗 2 点。
- 支付方式先用支付宝二维码占位，超级管理员手动加点。
- 注册增加图形验证码和邮箱验证码。
- 5 分钟内最多生成 3 份报告。
- 阿里云邮件服务接口预留并接入；开发环境可用日志验证码，生产环境缺配置时拒绝发码。

本阶段不实现：

- 自动支付回调。
- 微信支付。
- 发票、退款、订单对账。
- 复杂风控评分。

## 数据模型

新增或扩展：

- `User.creditBalance`：用户当前点数。
- `User.freeQuotaDate`：免费额度日期，格式 `YYYY-MM-DD`。
- `User.freeQuotaUsed`：当天已用免费次数。
- `UserSettings.serperApiKeyEncrypted`：账号级 Serper key。
- `UserSettings.enableSerperSearch`：是否启用账号级 Serper。
- `AppSetting`：管理员级模型、搜索、邮件、支付宝二维码配置。
- `PointTransaction`：点数流水。
- `PurchaseOrder`：点数订单，第一版手动确认。
- `VerificationCode`：注册邮箱验证码。
- `RateLimitEvent`：生成报告频率限制记录。

## 额度规则

生成报告前在服务端执行：

1. 用户必须登录。
2. 检查 5 分钟内生成报告次数，超过 3 次则拒绝。
3. 计算本次消耗：快速 1 点，深度 2 点。
4. 如果当天免费次数小于 3，则使用免费额度。
5. 免费额度使用管理员模型 API 与管理员搜索 API。
6. 免费额度用完后，扣用户点数；点数不足则拒绝并提示购买。
7. 生成失败不扣额度；只有模型返回并成功保存报告后才提交额度消耗。

## 搜索规则

搜索 key 来源：

- 免费额度：只使用管理员 Tavily / Serper key。
- 点数额度：优先使用用户账号 key；用户未配置时回退管理员 key。

供应商规则：

- 只配置 Tavily：只搜 Tavily。
- 只配置 Serper：只搜 Serper。
- 两者都配置：两个都搜并去重合并。
- 都未配置：报告必须标注未启用联网核验。

报告中必须输出来源链接、依据片段、支持的判断、核验状态和不能确认的原因。

## 注册验证码

注册流程：

1. 前端请求图形验证码。
2. 用户填写图形验证码。
3. 图形验证码通过后，后端发送邮箱验证码。
4. 用户提交邮箱验证码、邮箱、密码完成注册。

限制：

- 图形验证码 10 分钟有效。
- 邮箱验证码 10 分钟有效。
- 同一邮箱 10 分钟内最多发送 3 次。
- 同一 IP 10 分钟内最多发送 5 次。

## 邮件服务

邮件发送通过 `lib/email.ts` 抽象：

- `ALIYUN_ACCESS_KEY_ID`
- `ALIYUN_ACCESS_KEY_SECRET`
- `ALIYUN_DM_ACCOUNT_NAME`
- `ALIYUN_DM_FROM_ALIAS`
- `ALIYUN_DM_REGION`

生产环境未配置时，发送验证码接口返回“邮件服务未配置”。开发环境输出验证码到日志。

## Markdown 导出

导出内容顺序：

1. 报告标题和元信息。
2. 阅读价值结论。
3. 新闻简要总结。
4. 一句话审视结论。
5. 核心指数表格。
6. 最关键发现。
7. 联网核验依据。
8. 暂无法核验的问题。
9. 支持原文叙事的证据。
10. 主要信息缺口。
11. 观隅九镜审读法。
12. 用户提问与回答。
13. 新闻原文。

Markdown 不导出 JSON。

## 管理员能力

账号管理页超级管理员区增加：

- 查看/修改管理员 API 配置。
- 查看待确认订单。
- 为用户手动加点。
- 查看所有报告并删除。

第一版支付为手动确认，不做自动回调。
